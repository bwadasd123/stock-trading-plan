# Trading Operation Workflow

## 每日操作检查清单

### 开盘前 (09:25-09:30)
1. **获取最新价格** — 每次分析前必调用实时价格API
2. **检查集合竞价** — 高开<3%才做
3. **确认昨日扫描结果** — 读取`stock_scan_results`（不是`stock_snapshot`）

### 买入执行 (09:30-10:00)
```
09:35 第一笔: 3000元（约100-200股）
10:00 如回调2%，第二笔: 3000元
```

### 持仓期间 (每天收盘前)
- 盈利>15% → 卖出50%
- 亏损>8% → 全部卖出
- 持仓5天 → 不管盈亏都减仓

## 实时价格获取

**规则**: 每次给出操作建议前，必须先获取最新价格

```bash
# 获取最新价格
curl -s "http://push2delay.eastmoney.com/api/qt/stock/get?secid=0.{code}&fields=f43,f170,f44,f45,f46,f60" | python3 -c "import sys,json; d=json.load(sys.stdin)['data']; print(f'价格:{d[\"f43\"]/100} 涨跌:{d[\"f170\"]}%')"
```

## 仓位计算

| 本金 | 单股仓位 | 买入方式 | 止损 | 止盈 |
|------|----------|----------|------|------|
| 3万 | 6000元(20%) | 分两笔各3000元 | -8% | +15% |

## 实战教训

### 科安达(002972) 2026-06-17
- **错误**: 第一笔只买100股(1600元)，后追高加仓300股
- **结果**: 均价从16.22升至16.443，成本涨0.22元
- **教训**: 
  1. 第一笔必须买足3000元
  2. 回调2%再加第二笔
  3. 不要追高加仓

### 价格数据
- **错误**: 用旧价格给建议
- **纠正**: 用户说"你的现价不应该更新一下吗"
- **规则**: 每次分析前必须刷新价格

## 备份文件读取

**陷阱**: `stock_snapshot`有几百只股票，`stock_scan_results`通常只有1只

```python
import json
data = json.load(open('backup_YYYY-MM-DD_N.json'))
results = data.get('stock_scan_results', [])  # 用这个！
```
