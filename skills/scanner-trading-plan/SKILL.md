# 扫描结果交易策略

## ⚠️ 用户偏好
- **本金固定3万**：不要反复询问本金金额
- **单股操作**：每天通常只有1只扫描结果
- **简洁回答**：用户偏好直接给方案，不要过多解释
- **分析目标是赚钱**：不要只罗列指标数字，要结合走势、量能、主力行为综合判断能不能赚钱。用户原话："不单纯从指标 我的目的是赚钱"
- **回测验证**：系统已内置回测功能（侧边栏📊回测系统），可验证策略历史收益

## 基本参数
- **本金**: 30,000元
- **单股仓位**: 6,000元 (20%)
- **买入方式**: 分两笔，各3,000元
- **止损线**: -8% (-480元)
- **止盈目标**: +15% (+900元)
- **持仓时间**: 3-5天

## ⚠️ 读取扫描结果的陷阱（重要！）

**我犯过的错误**：读取`stock_snapshot`得到345只股票，误以为没有明确推荐。实际上`stock_scan_results`只有1只股票。

备份JSON文件包含**两个数组**，含义完全不同：
- `stock_snapshot` — 当天全量扫描到的股票（几百只），是未筛选的原始数据
- `stock_scan_results` — **经过筛选的推荐股票**（通常1只），这才是你要操作的

**错误示范**：读`stock_snapshot`得到345只股票 → 误以为没有明确推荐
**正确做法**：读`stock_scan_results`，通常只有1只股票

```python
import json
data = json.load(open('backup_YYYY-MM-DD_N.json'))
results = data.get('stock_scan_results', [])  # ← 用这个！
```

## 每日操作流程

### 扫描出结果当天 (17:00后)
1. 分析技术指标 (RSI、量比、MACD)
2. 决定是否操作
3. 准备资金6,000元

### 次日执行
```
09:25 看集合竞价 (高开<3%才做)
09:30 开盘观察
09:35 买入3,000元
10:00 如回调2%，再买3,000元
```

### 持仓期间 (每天收盘前检查)
- 盈利>15% → 卖出50%
- 亏损>8% → 全部卖出
- 持仓5天 → 不管盈亏都减仓

## RSI与持仓时间对照表

| RSI区间 | 持仓时间 | 目标收益 | 止损线 |
|---------|----------|----------|--------|
| 70-75 | 3-5天 | +15% | -8% |
| 75-80 | 2-3天 | +10% | -6% |
| >80 | 1-2天 | +8% | -5% |

## 收益预期 (每月20次操作)
- 胜率60%: 月赚约7,000元 (+23%)
- 保守估计: 月赚3,000-5,000元 (+10-17%)
- 标准估计: 月赚5,000-8,000元 (+17-27%)

## 风险控制
- 每周止损上限: -2,400元 (-8%)
- 每月止损上限: -5,000元 (-17%)
- 达到止损线后停止操作3天，复盘总结

## 买入条件检查清单
- [ ] RSI > 70 但 < 80
- [ ] MACD金叉确认
- [ ] 量比 > 1.5
- [ ] 价格站上MA20
- [ ] 板块资金流入
- [ ] 高开幅度 < 3%

## 卖出信号
1. 达到止盈目标 → 分批卖出
2. 触发止损线 → 立即清仓
3. RSI>80且顶背离 → 果断卖出
4. 持仓5天仍未盈利 → 减仓50%

## ⚠️ 实战教训（2026-06-17 科安达复盘）

**用户实际操作**：先买100股(成本16.22)，后看到大手笔挂单加仓300股，最终400股均价16.443
**问题**：
1. 第一笔买太少（100股≈1600元，应买3000元）
2. 加仓抬高成本（16.22→16.443，涨了0.22元）
3. 追高加仓（看到大单信号时已涨不少）

**教训**：
- **第一笔必须买足3000元**，不要保守只买一半
- **回调2%再加第二笔**，不要追高加仓
- **看到信号≠立即行动**，要评估价格位置

## ⚠️ 价格数据陷阱

**用户纠正**："你的现价不应该更新一下吗，要去取最新的啊"

**规则**：每次给出操作建议前，**必须先获取最新价格**，不能用之前缓存的数据。

```bash
# 获取最新价格（每次分析前必做）
curl -s "http://push2delay.eastmoney.com/api/qt/stock/get?secid=0.002972&fields=f43,f170,f44,f45,f46,f60" | python3 -c "import sys,json; d=json.load(sys.stdin)['data']; print(f'价格:{d[\"f43\"]/100} 涨跌:{d[\"f170\"]/100}%')"
```

## 价格监控（企业微信群推送）

**用户偏好**：不要只推触发条件，要**频繁推送**当前状态。用户说"感觉这样推送的有点少"。

**企业微信Webhook**：问用户要webhook地址，格式 `https://qyapi.weixin.qq.com/cgi-bin/webhook/send?key=xxx`

**监控脚本**：`~/.hermes/profiles/eastmoney-bot/scripts/price_monitor.py`
- 整数涨跌幅提醒（-10%到+10%每个整数百分比）
- 定时汇报（每小时推一次当前价格盈亏）
- 止盈止损触发推送
- 开盘收盘（9:30开盘价，15:00收盘总结）
- 双重价格对比（扫描发现价 vs 买入价）

**创建CronJob**：
```
schedule: */5 9-15 * * 1-5  # 交易日每5分钟
no_agent: true
deliver: origin
```

**CronJob script必须放在** `~/.hermes/profiles/eastmoney-bot/scripts/` 目录下，否则找不到文件。

**推送内容格式**：
```
📊 科安达 现价17.51 (+6.5%) 涨跌+1.27%
持仓400股 均价16.443 盈利+427元
止盈18.91 止损15.13
```
