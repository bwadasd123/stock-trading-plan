# 回本买入价 + target_buy 自动提醒

## 回本买入价公式

**每次卖出后必须计算并告知用户：**

```
回本买入价 = 卖出均价 - 已实现亏损 / 目标股数
```

例：芯片ETF 159599 — 卖出均价3.694，亏损-328.19，目标2600股
→ 回本买入价 = 3.694 - 328.19/2600 = 3.694 - 0.126 = **3.568**（实际用3.67）

## T接回实际成本

⚠️ **必须摊入之前亏损！**

```python
# T接回实际成本 = 买入价 + |之前累计亏损| / 新股数
# 例：之前亏328，2600@3.619接回 → cost = 3.619 + 328/2600 = 3.746
# ❌ 错误：直接用3.619 → 止损偏宽，用户一眼发现
# ✅ 正确：用3.746 → 真实持仓成本
```

## target_buy 字段

观察股票可设`target_buy`，到达自动推送：

```python
{
    "type": "观察",
    "target_buy": 3.67,  # 回本买入价
}
```

monitor的main()循环优先检查此字段，高于普通买入信号。

## ⚠️ 条件单触发≠无条件执行（2026-07-13血泪教训）

**亚威股份案例**：11.00条件单触发时深证已跌超-2%，但没刹一脚直接执行。买入后当日继续跌到10.78，亏-252元。

**规则**：
- 条件单触发时，必须检查大盘状态
- 大盘-2%以上 → 暂停，等大盘企稳再手动判断
- 大盘-3%以上 → 直接取消，不执行
- 不能因为"挂了条件单"就无脑让系统触发

**条件单执行前置检查**：
```
1. 触发价到达？ → YES
2. 大盘 < -2%？ → YES → 暂缓
3. 板块 < -3%？ → YES → 暂缓
4. 两者都OK → 执行
```

## ⚠️ 底部反弹后不追高（2026-07-13教训）

**沃顿科技案例**：11.18→13.48，两天反弹+20.6%，MA5=12.22，现价高悬MA5之上。设target_buy=13.33(MA10)看似有道理，但实际盈亏比崩了：
- 上方MA20=13.85仅4%空间
- 下方止损12.26要扛9%
- 盈亏比 1:2.25，完全不划算

**规则**：底部反弹超过15-20%后，即便RSI不极端，MA10/MA20回踩空间太小，不再设target_buy。必须等回踩MA5或确认突破MA20站稳后重新评估。

## 用户偏好

- 只告诉一个价格，不展开分析
- 格式：`🔁 回本买入价: ≤ X.XX（N股）`
- 推送消息中自动带上
