# 涨停持有分析工作流

## ⚠️ 核心规则（2026-07-13确立）

**涨停不是无脑卖！** 用户明确要求：涨停先不卖，发给AI分析后再决定。

## 工作流（30秒分析）

```
1. 封板量   — 买一封单量、封单金额，是否硬封
2. 板块资金 — 所属板块是否当天主线、资金流入
3. 第几个板 — 首板大概率溢价，连板风险高
```

## 判断标准

| 条件 | 操作 |
|------|------|
| 首板 + 封单硬(>20万手) + 主线板块 | ✅ 持有，等次日溢价 |
| 连板(第2-3板) + 封单一般 | ⚠️ 考虑减仓 |
| 烂板(反复开板) + 非主线 | ❌ 卖出 |

## 案例

- 沃顿科技 7/13：首板封死@14.03，封单37.6万手。但用户已清仓无法享受。
- 教训：7/11(周五)涨停@12.60时应该用此流程分析，可能就不会卖飞10%。

## 触发方式

用户说"又涨停了" → 立即执行30秒三步分析 → 给结论。
不需要用户再说"帮我分析一下"。
