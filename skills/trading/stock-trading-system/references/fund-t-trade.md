# 基金做T降本策略

## 核心原理

利用基金**先进先出(FIFO)**规则：
1. 大跌日买入（低净值）→ 增加份额
2. 反弹后赎回最早批次（高成本）→ 等额减少份额
3. 净效果：总份额不变，成本净值下降

## 前置条件

- ✅ C类基金持有超7天（赎回费0%）
- ✅ 早期批次成本明显高于当前净值
- ✅ 有闲置资金做T
- ⚠️ 周五买入锁仓到下周一确认份额

## 操作步骤

### 1. 确认各批次成本

```python
# 例：015690基金
# 6/29 5000 成本 ~8.3
# 7/1  5000 成本 ~8.1
# 7/2  5000 成本 ~7.8
# 加权成本 8.1174 × 1847.88份
```

### 2. 计算做T收益

```python
buy_price = 7.086    # 今日估值
buy_amount = 5000     # 做T金额
buy_shares = buy_amount / buy_price  # 新份额

# 场景：反弹到不同价位
for sell_price in [7.20, 7.35, 7.50]:
    profit = buy_shares * (sell_price - buy_price)
    # 7.20 → +80, 7.35 → +186, 7.50 → +292
```

### 3. 赎回执行

- 反弹日15:00前提交赎回
- 赎回金额 = 做T买入份额 × 当日净值（确保总份额不变）
- 基金先进先出 → 自动赎回最早（成本最高）批次
- T+1确认份额，T+2资金到账

### 4. 降本后新成本

```python
# 做T前
old_cost = 8.1174
old_shares = 1847.88
old_total = 15000

# 做T买入
new_shares = 706  # 5000 / 7.086
# 反弹赎回（等额）
# 新成本 = (old_total + 5000 - 赎回金额) / old_shares
# 约 7.92（净值反弹到7.35时）
```

## 风险

1. **净值继续跌**：买在7.086但下周跌到6.80 → 新买的也亏，T失败
2. **反弹不及预期**：只涨到7.10 → T收益仅17元，不够覆盖摩擦成本
3. **流动性风险**：赎回T+2才到账，资金锁定2天

## 决策框架（2026-07-17确立）

```
基金单日-6% + 大盘暴跌 → 不问"要不要等企稳"
                        → 直接算做T收益+新成本
                        → 用户判断是否执行

个股-6% + 大盘暴跌 → 必须等企稳（可能继续跌停/暴雷）
```
