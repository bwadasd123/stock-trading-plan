# JMY科技 A股监控系统操作逻辑

## 🔄 整体流程

```
扫描系统 → 发现信号 → 决策买入 → 监控系统 → 自动提醒 → 决策卖出
```

---

## 1️⃣ 扫描系统 (stock-screener)

- **位置**: `/home/jmy/stock-screener/`
- **运行**: 每天收盘后自动扫描全市场
- **功能**: 筛选符合条件的股票（RSI、涨幅、量能等）
- **存储**: MySQL 数据库
- **前端**: http://localhost:8080/report
- **输出**: 信号列表（代码、名称、发现价、RSI）

---

## 2️⃣ 决策环节 (用户)

- 看扫描结果 → 判断是否买入
- 告知系统: "买入XX，价格XX，股数XX"
- 系统更新监控配置

---

## 3️⃣ 监控系统 (price_monitor.py v14)

- **位置**: `/home/jmy/.hermes/profiles/eastmoney-bot/scripts/price_monitor.py`
- **运行**: Cronjob 每分钟执行（交易时间 9:00-15:00）
- **配置**:
  - 总资金: 32,319元（3万本金 + 盈利2,319元）
  - 单股仓位: 20% = 6,464元
  - 持仓上限: 5个交易日
  - 默认止盈: +15%
  - 默认止损: -8%

### 监控内容

| 功能 | 说明 |
|------|------|
| 🔴🟢 涨停跌停 | 状态变化多次触发（涨停/打开/再涨停） |
| 📈📉 逼近涨跌停 | 涨跌幅≥8%时提醒 |
| 📈 整数涨跌幅 | -10%到+10%每个整数提醒 |
| 💰 成本线提醒 | 0.5%间隔提醒 |
| 🎯 止盈止损 | 触发自动提醒 |
| ⏰ 持仓天数 | 超5天自动提醒 |
| 📊 整点播报 | 含持仓分析和建议 |

### 输出
- 企业微信推送

## 🎯 用户操作流程

1. **看报告** → http://localhost:8080/report
2. **决策** → 是否买入
3. **告知系统** → 买入价格和股数
4. **等提醒** → 系统自动推送
5. **卖出** → 告知系统，清空配置
